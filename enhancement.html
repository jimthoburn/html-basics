It’s easier to build in layers

Content first

Start by deciding what features and content you want to have. Add them to your document as plain text.

http://amazon.com/hobbes.jpg

Van Gogh would’ve sold more than one painting if he’d put tigers in them. –Hobbes

Mark up your content to give meaning to the text and indicate relationships.

<figure>

  <img src="http://amazon.com/hobbes.jpg" alt="" />

  <figcaption>Van Gogh would’ve sold more than one painting if he’d put tigers in them. –Hobbes</figcaption>

</figure>

And then style

Use styles to move the markup in the direction of your design.

<style>

figure {
  background: black;
  padding: 1em;
}

img {
  border: 1px solid white;
}

figcaption {
  margin-top: 1em;
  color: white;
}

</style>

Use CSS for mouse overs, and be sure the content is still readable without them.

<style>

body:hover figcaption {
  visibility: hidden;
}

figure:hover figcaption {
  visibility: visible;
}

</style>

Test along the way to see if the content and page features work well on a few different size devices.

Browser Stack will be useful for this stage.

And lastly scripting

Add JavaScript as needed for special interaction and features that aren’t available using HTML and CSS.

<script>

document.querySelector(".play").addEventListener("click", function(e) {
  startSound();
  e.preventDefault();
});
document.querySelector(".stop").addEventListener("click", function(e) {
  stopSound();
  e.preventDefault();
});

</script>

Try to leave things out of the markup that aren’t part of the core content. Keep it simple.

If something requires JavaScript to operate and isn’t essential (there are other ways to accomplish the task), use JavaScript to add it later.

<script type="text/template" id="audio-controls">

<ul class="controls">
  <li class="play"><a href="#play">Play</a></li>
  <li class="stop"><a href="#stop">Stop</a></li>
</ul>

</script>

Test along the way with JavaScript turned off to see your content and features are still readable.
